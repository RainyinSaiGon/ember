## Step 1 — Resolve context

- Current branch: `git rev-parse --abbrev-ref HEAD`. If it's `main`, stop.
- Base branch: `main` (hardcoded).
- Linked issue: parse `#<n>` from the branch name (e.g. `feature/#12-resp-parser` → 12);
  else ask the user. Use it for `Closes #<n>`.
- Push the branch if it has no upstream:
  ```bash
  git push -u origin <branch>
  ```

## Step 2 — Build the PR body

Read the diff:
```bash
git diff main...HEAD --stat
git diff main...HEAD
```

Fill the standard PR structure with real content — no placeholder comments left:

```markdown
## Summary
<What changed and why, drawn from the diff and commits.>

## Related
Closes #<n>

## What Changed
- [ ] Core server / networking
- [ ] RESP parser
- [ ] Command dispatcher / keyspace
- [ ] Persistence (RDB / AOF)
- [ ] Replication
- [ ] Data types
- [ ] Tests
- [ ] Build / config
- [ ] Documentation

## Verification
<Exact commands: `./gradlew build`, `./gradlew test`, plus any manual redis-cli steps.>

## Breaking Changes
<Yes / No, and what.>

## Follow-Up
<Items intentionally deferred.>
```

Tick only the checkboxes the diff actually touches.

Write the body to `.claude/drafts/draft-pr.md` with frontmatter:
```markdown

title: "<title>"

issue: <n>

<body>
```

## Step 3 — Resolve title & labels

- **Title:** sentence-case prose. Default to the issue title (from `gh issue view <n> --json title`); offer an alternative if it doesn't read well as a PR title.
- **Labels:** take from the issue (`gh issue view <n> --json labels`); keep only those in the known label list above.
- **Draft:** default to ready (not draft) unless the user says otherwise.

## Step 4 — Confirm, then create

Show: title, `main ← <branch>`, `Closes #<n>`, labels, draft/ready, short body preview. Get a go-ahead. Then:

```bash
# Strip YAML frontmatter before passing to --body-file
tmp=$(mktemp)
awk 'BEGIN{fm=0} /^---[[:space:]]*$/{fm++; next} fm>=2{print}' \
  .claude/drafts/draft-pr.md > "$tmp"

gh pr create \
  --repo RainyinSaiGon/ember \
  --base main \
  --head <branch> \
  --title "<title>" \
  --body-file "$tmp" \
  [--label "<label>"] \
  [--draft]

rm -f "$tmp"
```

Do **not** include `--reviewer` or `--assignee`.

## Step 5 — Report back

- Print the PR **URL**.
- Remind the user to add reviewers themselves.
- Offer follow-ups (don't run without asking): `gh pr view --web`, `gh pr ready`, or delete `.claude/drafts/draft-pr.md`.
